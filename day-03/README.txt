# Day 3 – Role-Based Prompting

## 📌 Overview

Today I explored **Role-Based Prompting**, a prompt engineering technique where you assign an AI a specific role before asking a question. This helps Claude generate responses based on the expertise and perspective of that role.

---

## 🎯 Objective

- Understand Role-Based Prompting
- Compare responses with and without assigned roles
- Learn how different personas influence AI output
- Document the results

---

## 🧠 What I Learned

- AI provides more focused responses when given a role.
- Different personas produce different perspectives for the same question.
- Role prompting improves clarity, depth, and usefulness.
- A simple role assignment can significantly enhance prompt quality.

---

## 🛠️ What I Used

- Claude AI
- GitHub
- Prompt Engineering techniques

---

## 📝 Prompts Used

### Without Role

> How can I improve my GitHub profile as a Computer Science student?

---

### Founder Persona

> You are a startup founder who has hired many software engineers.

> How can I improve my GitHub profile as a Computer Science student?

---

### Developer Persona

> You are a senior software developer with 10 years of experience.

> How can I improve my GitHub profile as a Computer Science student?

---

## 📊 Comparison

### Without Role
- Generic suggestions
- Broad advice
- Less personalized

### Founder
- Focus on portfolio
- Hiring perspective
- Real-world impact

### Senior Developer
- Code quality
- Project architecture
- Best practices
- Technical growth

---

## 🚀 Key Takeaways

✅ Better prompt quality

✅ More personalized responses

✅ Expert-level guidance

✅ Less follow-up prompting

## 💡 Conclusion

Role-Based Prompting is one of the simplest yet most effective prompt engineering techniques. By assigning Claude a professional role, the same question can produce more relevant, detailed, and actionable responses.
